# Анализ сложности алгоритма линейного поиска

## Описание алгоритма

Линейный поиск последовательно перебирает каждый элемент списка, сравнивая его с искомым значением. Поиск завершается при первом совпадении или после проверки всех элементов.

## Временна́я сложность (Time Complexity)

| Случай | Сложность | Пояснение |
|--------|-----------|-----------|
| Лучший (Best) | **O(1)** | Искомый элемент — первый в списке |
| Средний (Average) | **O(n)** | В среднем приходится просматривать n/2 элементов |
| Худший (Worst) | **O(n)** | Элемент стоит последним или отсутствует |

**Вывод:** В общем случае говорят, что временна́я сложность линейного поиска — **O(n)**, поскольку она линейно растёт с размером входных данных.

## Пространственная сложность (Space Complexity)

**O(1)** — алгоритм использует только одну вспомогательную переменную (индекс/итератор). Объём дополнительной памяти не зависит от размера списка.

## Выводы

1. **Простота реализации.** Линейный поиск не требует предварительной сортировки данных и работает с любой структурой, поддерживающей итерацию.
2. **Масштабируемость.** При увеличении списка вдвое время поиска также увеличивается примерно вдвое — это и есть линейная зависимость O(n).
3. **Применимость.** Линейный поиск оправдан для небольших или несортированных наборов данных. Для больших сортированных коллекций предпочтительнее бинарный поиск O(log n).
